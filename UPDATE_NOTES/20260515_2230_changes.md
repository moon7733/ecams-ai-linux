## 수정 3: contextBuilder의 servlet 우선순위 강화

### 문제
- #2 (Haiku): "체크아웃취소 이력" 질문에 컨텍스트로 `Cmr0101Servlet`이 들어가지 않은 것으로 추정
- 기존 `rankServlets`는 common/util만 후순위로 미는 단순 정렬 — JS가 실제로 직접 호출하는 servlet을 보장하지 못함
- 예: CheckOutCnl.js의 `reqQuestConf()`가 부르는 `Cmr0101Servlet`이 Graph 파일 순서에 따라 컷오프될 수 있음

### 원인
- (B) contextBuilder 우선순위 — JS 함수가 직접 호출하는 핵심 servlet이 후보 정렬에서 밀려남
- Graph/Files는 "참조 관계"를 보여주지만 "직접 호출"과 "의존 전이"를 구분 못 함

### 수정 내용
- **파일**: `contextBuilder.js`
- **신규 함수**: `extractDirectServlets(jsWikiContent)` — JS wiki의 `[[Servlets/<name>|...]]` 패턴 추출 (= JS가 직접 부르는 servlet 목록)
- **rankServlets 시그니처 변경**: `(servletFiles, directServlets = new Set())`
- **새 정렬 우선순위**:
  1. JS가 직접 호출하는 servlet (`directServlets` 포함)
  2. 도메인 prefix (`/ecmr/`, `/ecmm/` 등)
  3. common/util (`_common_`, `_util`)
- **buildContext 호출 변경**: `rankServlets(extractServletFilenames(graphContent), directServlets)`
- **효과 기대**: 사용자 질문과 매칭된 JS의 메인 servlet wiki가 항상 컨텍스트 상위에 포함됨
