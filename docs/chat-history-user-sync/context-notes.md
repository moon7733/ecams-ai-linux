# Chat History User Sync Context Notes

## 결정 1 (2026-07-02) — 서버 저장소는 유저별 JSON 파일

- 대화기록은 `logs/chat_history/<userId>.json`에 저장한다.
- 이유. 현재 앱이 파일 기반 설정(`users.json`, `repos.json`)을 쓰고 있어 별도 DB 도입 없이 가장 작게 해결된다.
- 제한. 파일 저장은 단일 서버 프로토타입에 적합하며, 다중 서버 배포 시 공유 저장소가 필요하다.

## 결정 2 (2026-07-02) — localStorage는 캐시로 강등

- 브라우저 기록은 유지하되 서버 기록과 병합한다.
- 이유. 기존 사용자의 기록을 버리지 않고, 서버 장애나 일시적인 네트워크 실패 때도 화면 경험을 유지한다.

## 결정 3 (2026-07-02) — 화면 삭제와 감사 로그는 분리

- `DELETE /api/chat/history/:id`는 화면 대화기록만 삭제한다.
- 이유. `answer_log` 계열 감사 기록은 운영 추적용이므로 사용자의 화면 정리 동작과 결합하지 않는다.

## 결정 4 (2026-07-02) — 개별 삭제 단위는 질문+답변 turn 쌍

- 사용자 확정에 따라 개별 메시지 삭제는 질문 하나만 지우지 않고, 해당 사용자 질문과 바로 다음 AI 답변을 함께 삭제한다.
- 이유. 질문만 삭제하면 답변 맥락이 떠서 대화 흐름이 깨진다.
- 진행 중 응답이 있을 때는 삭제를 막는다. 스트리밍 중 DOM과 `messages` 배열이 엇갈리는 것을 피하기 위해서다.
- 마지막 turn을 삭제해 대화가 비면 화면 기록의 해당 대화도 제거한다. 감사 로그는 계속 보존한다.
