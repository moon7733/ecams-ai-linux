# 답변 포맷 경직성 완화 계획

## 목적

신규 설계·정책 정리·문서화 요청이 기존 로직 분석 포맷에 억지로 매핑되어 답변 품질이 떨어지는 문제를 줄인다.

## 방향

- 개발자/admin용 `SYSTEM_PROMPT`에 설계/신규 요구용 형식 C를 추가한다.
- 기존 `## 0. 분석 근거` 앵커는 유지해 `stripAgyPreamble` 소스누출 방어와 호환한다.
- 파일 생성이 불가능한 AGY 응답 환경에서 "md 파일을 만들었다"고 말하지 않도록 막고, 문서 산출물은 답변 본문에 Markdown으로 제공하게 한다.
- customer/enduser는 기존 화면 조작 안내 원칙을 유지하되, 문서 요청 시 파일 생성 약속은 하지 않게 한다.

## 검증

- `node --check server.js`로 문법 오류가 없는지 확인한다.
- 프롬프트 문자열 안에 형식 C와 파일 생성 금지 지시가 포함됐는지 확인한다.

## 추가 — 형식 C 신규 파일 첨부(다운로드/미리보기) (2026-07-02)

사용자가 형식 C에서 "신규 파일/설계안을 만들어달라"고 하면 실제 다운로드·미리보기가 가능해야 한다는 요구.

- **저장 위치 결정.** repo/디스크에는 쓰지 않는다. AGY가 프롬프트 지시만으로 절대경로 밖 쓰기를 지키지 않는다는 게 이미 동시성 이슈에서 실증됐다(`to-codex.md` "Claude 회신 3" — `--sandbox`조차 Write/Edit 비-terminal 도구의 절대경로 쓰기를 못 막음). 대신 답변 텍스트 안에 완성된 파일 내용을 담고 클라이언트에서만 다운로드/미리보기한다.
- **히스토리 지속성은 별도 저장 없이 해결됨.** `messages.push({role:'ai', content: fullAnswer, ...})`가 답변 전문을 그대로 저장하고, `loadChat`이 재로드 시 동일한 렌더 파이프라인을 다시 태우므로 코드블록 안의 신규 파일 내용도 그대로 다시 보인다. 대화 삭제 시 자동으로 같이 지워지는 것도 부가 이득.
- **모델 출력 규약.** `[📄 파일명](newfile:///상대경로)` 링크 바로 뒤에 코드블록을 붙이는 형식(서버 `SYSTEM_PROMPT` 형식 C "신규 파일 첨부 규칙" 섹션).
- **프론트 파싱.** 기존 코드블록 정규식은 건드리지 않고, `newfile:///` 마커+코드블록을 먼저 별도로 추출하는 패스를 앞에 추가(`formatContent`, `public/index.html`). 마커가 없거나 형식이 흐트러지면 평범한 코드블록으로 그레이스풀 디그레이드.
- **뷰어 재사용.** 소스뷰어는 `svCurrentBase64`/`svCurrentFilename` 상태만 채우면 서버 호출 없이 그대로 렌더된다는 걸 확인 — `svOpenVirtualFile()`이 `svReadFile()`의 서버 fetch만 건너뛰고 나머지(다운로드·AI분석·CodeMirror 렌더)는 그대로 재사용.
