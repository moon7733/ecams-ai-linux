# 답변 포맷 경직성 완화 작업 기록

## 2026-07-02

- 사용자 보고와 `to-codex.md`를 비교했다. 이전 Codex 답변과 Claude 제안은 큰 방향이 같다. 둘 다 포맷 강제 자체보다 의도별 분기가 부족한 것이 문제라고 본다.
- 최종 판단은 Claude안에 가깝게 정했다. 이유는 `stripAgyPreamble`가 `## 0. 분석 근거`와 `1. 한 줄 요약`을 소스누출 제거 앵커로 사용하고, enduser 렌더 개선도 고정 제목에 의존하기 때문이다. 완전 자유화는 안전장치와 렌더링 안정성을 동시에 흔들 수 있다.
- 개발자/admin용 답변에는 형식 C를 추가한다. 설계/신규 요구 질문은 기존 실행 흐름·로직 상세 섹션을 억지로 채우지 않고 요구사항, 현행 구조, 설계안, 변경 대상, 미결정 사항으로 답하게 한다.
- `md 파일로 만들어줘` 요청은 실제 파일 생성 약속으로 처리하지 않는다. eCAMS AI 화면에서 생성 파일을 열거나 다운로드할 경로가 없으므로, 완성된 Markdown 문서 본문을 채팅에 포함하는 방식이 현재 제품 구조에 맞다.
- customer/enduser는 계속 화면 조작 안내 persona를 유지한다. 다만 파일 생성 약속 금지는 공통으로 적용한다.

## 2026-07-02 (후속) — 형식 C 신규 파일 첨부

- 사용자 요청. 형식 C에서 신규 파일/설계안을 만들어달라고 하면 실제로 다운로드·미리보기가 되게 해달라.
- **디스크 저장은 채택 안 함.** 사용자가 "AGY한테 지정 폴더에만 만들라고 지시하면 안 되나"를 물었는데, 이미 이 레포의 동시성 이슈 조사에서 `--sandbox` 같은 코드 레벨 제약조차 AGY의 비-terminal Write/Edit 절대경로 쓰기를 못 막았다는 게 실증됐다(`to-codex.md` "Claude 회신 3"). 프롬프트 지시(자연어)는 그보다 약한 통제라 신뢰 불가 — 답변 텍스트 전달(클라이언트 전용 처리)만 채택.
- **히스토리 걱정은 기우였다.** `messages.push({role:'ai', content: fullAnswer})`가 답변 전문을 저장하고 `loadChat`이 동일 파이프라인으로 재렌더하므로, 별도 저장 없이도 신규 파일 코드블록이 히스토리에서 그대로 다시 보이고 버튼도 재생성된다. 대화/메시지 삭제 시 자동으로 같이 지워지는 것도 부가 이득(요구사항과 일치).
- **구현 완료.** `server.js` SYSTEM_PROMPT 형식 C에 `newfile:///` 링크+코드블록 컨벤션 추가. `public/index.html`은 기존 코드블록/diff 정규식을 건드리지 않고 `newfile:///` 마커를 먼저 추출하는 별도 패스만 추가(마커 없거나 형식이 흐트러지면 평범한 코드블록으로 그레이스풀 디그레이드). 뷰어는 `svCurrentBase64`/`svCurrentFilename` 상태만 채우면 서버 호출 없이 기존 소스뷰어(다운로드·AI분석·CodeMirror)가 그대로 재사용된다는 걸 확인해 `svOpenVirtualFile()`로 구현.
- **검증.** `node --check server.js` 통과, 인라인 스크립트 문법 검사 통과, 격리 스크립트로 정규식 추출 로직 5개 시나리오(정상/공백변형/마커뒤텍스트/마커없음/다중파일) 검증. 실사용(실제 채팅에서 버튼 클릭) 검증은 배포 후 대기.
