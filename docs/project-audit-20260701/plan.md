# eCAMS AI 프로젝트 감사 계획

## 목적

현재 eCAMS AI 프로젝트 구조와 AI 호출 경로를 파악하고, 실제 사용 중인 AGY 경로, Claude Code printmode 사용 범위, 모델 전환 가능성을 확인한다.

## 범위

- 현재 프로젝트 구조와 주요 모듈 파악.
- `agy` 기반 AI 질문 경로 확인.
- Claude Code printmode가 소스뷰어의 소스분석에만 쓰이는지 확인.
- 기존 3.5 Flash 계열 모델 사용 위치 확인.
- Codex 사용을 고려한 OpenAI GPT 계열 exec 방식 도입 가능성 검토.
- API 키 방식은 무료 Gemini 키의 되묻기와 임베딩 보조에만 남기고, 그 외 유료 API 키 경로는 삭제 후보로 분류.
- 발견한 문제점과 개선 우선순위 정리.

## 운영 제약

- 유료 API 키 기반 모델 호출은 사용하지 않는다.
- 현재 허용되는 API 키 방식은 AGY 답변 전 되묻기와 임베딩 보조에 쓰는 무료 Gemini 키뿐이다.
- GPT 계열 모델을 쓰더라도 OpenAI API 키를 직접 넣지 않는다.
- GPT 계열 후보는 Codex 또는 구독 세션으로 인증된 CLI/exec 방식만 검토한다.
- exec 방식이 월구독 범위 안에서 동작하는지는 구현 전에 별도 probe로 확인한다.

## 검증 방법

- `MEMO.md`와 관련 `docs/` 진행 문서 확인.
- `rg`로 AI 호출, 모델명, AGY, printmode, exec 경로 검색.
- `package.json`과 서버 진입점 확인.
- 필요 시 기존 `graphify-out` 자료를 참고한다.
- OpenAI 모델 정보는 공식 문서 기준으로 확인한다.

## 제외 범위

- 이번 단계에서는 코드 수정은 하지 않는다.
- 모델 전환 구현은 별도 합의 후 진행한다.
- 기존 미커밋 변경사항은 건드리지 않는다.
