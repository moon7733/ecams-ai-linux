# 플러그인에서 프로그램 신규 등록 시 경로 생성 흐름 상세 분석

## 1. 전체 흐름도

```
[플러그인 클라이언트]
  ↓
  RegistFileNewJob.java - 프로그램 정보 수집 및 서버 전송
  ↓
[플러그인 서버]
  ↓
  RegistFileNewService.java - 요청 처리
  ↓
  FileService.registAllCheckInFile() - 프로그램 경로 생성
  ↓
  PathService.getPathCD() - 경로 코드 생성 및 DB 저장
```

---

## 2. 단계별 상세 분석

### 2-1. 플러그인 클라이언트 (RegistFileNewJob.java)

**역할**: 사용자가 선택한 프로그램 파일 정보를 수집하여 서버로 전송

**경로 관련 정보 수집 (161줄)**:
```java
pathinfo_builder.setRelativitePath(resources.get(i).get("dirpath").toString());
```

**수집되는 정보**:
| 정보 | 변수명 | 출처 | 설명 |
|------|-------|------|------|
| 상대경로 | dirpath | resources 맵 | 베이스 경로 기준의 상대 경로 |
| 프로그램명 | rsrcname | resources 맵 | 파일 이름 |
| 프로그램 종류 | rsrccd | resources 맵 | 리소스 타입 코드 (확장자 기반) |
| 프로그램 타입 | pgmtypecd | resources 맵 | 프로그램 유형 코드 |
| 업무 코드 | jobcd | resources 맵 | 업무 식별 코드 |
| 변경 사유 | story | resources 맵 | 신규 등록 사유 |
| SR ID | srid | resources 맵 | 서비스 요청 ID (선택) |

**특징**:
- 이미 플러그인에서 상대경로(dirpath)를 보유
- 프로그램 종류(rsrccd)는 **사전에 결정되어 전송됨** (클라이언트에서 이미 판단)

### 2-3. 파일 경로 및 프로그램 종류 처리 (FileService.registAllCheckInFile)

#### 2-3-1. 프로그램 종류 결정 로직 (1222~1240줄)

**프로그램 종류가 하드코딩되지 않는 방식**:

```java
String rsrccd = null;
rsrccd = newList.get("rsrccd").toString();  // ← 입력값 받음
int j=0, k=0;

// ResourceType 테이블에서 모든 프로그램 종류 조회
List<ResourceType> rsuList = 
  (List<ResourceType>) resourceTypeDAO.getRsrcInfo(newList.get("syscd").toString());

if(null == rsrccd) {
  // rsrccd가 없으면 파일명의 확장자로 판단
  for (j=0; j<rsuList.size(); j++) {
    if(rsuList.get(j).getResourceExeName() != null) {
      // 예: ".java,.class,.jar" 형태의 확장자 목록 분석
      String[] exename = rsuList.get(j).getResourceExeName().split(",");
      
      for (k=0; k<exename.length; k++) {  
        // 파일명과 확장자 비교 (대소문자 무시)
        if(newList.get("rsrcname").toUpperCase().lastIndexOf(exename[k].toUpperCase()) > 0
            || newList.get("rsrcname").toUpperCase().equals(exename[k].toUpperCase())) {
          rsrccd = rsuList.get(j).getResourceType();  // ← 자동 결정
          break;
        }
      }
    }
  }
}
```

**특징**:
- ✅ **완전히 동적**: 프로그램 종류별 확장자는 DB(ResourceType) 테이블에서 관리
- ✅ **유연한 변경**: 신규 프로그램 종류 추가 시 코드 수정 불필요 (DB 추가만 필요)
- ✅ **확장자 기반**: .java → Java, .sql → SQL 등 자동 판정

**ResourceType 테이블 구조** (예):
```
rsrccd | rsrcname      | resourceExeName | resourceTypeName
-------|---------------|-----------------|----------------
R01    | Java Source   | .java,.class    | 자바 소스
R02    | SQL Script    | .sql            | SQL 스크립트
R03    | C/C++ Source  | .c,.cpp,.h      | C/C++ 소스
R04    | Config File   | .xml,.properties| 설정 파일
```

### 2-4. 경로 코드 생성 및 베이스 경로 조합 (PathService.getPathCD)

**베이스 경로 조회** (PathService.java 31줄):

```java
// cmm0102 테이블에서 베이스 경로 조회
// SYSHOMEPATH 함수 호출 + job명 추가
String basepath = pathDAO.getBasePath(nparam);
// 예 결과: "/home/ecams/system/KB001/J001"
```

**PathMapper.xml에 정의된 SQL** (24~28줄):
```xml
<select id="getBasePath" parameterType="hashmap" resultType="String">
  select SCMOWN.SYSHOMEPATH(#{cm_syscd}, #{cm_rsrccd}) || '/' || cm_jobname
  from cmm0102
  where cm_jobcd = #{cm_jobcd}
</select>
```

**설명**:
- `SCMOWN.SYSHOMEPATH()`: Oracle 저장 함수 (시스템 + 리소스별 기본 경로 반환)
- `/` + `cm_jobname`: 업무명 추가
- 결과 예: `/home/ecams/KB001/J001/MyJob`

---

**경로 조합 로직** (PathService.java 37~46줄):

```java
String dirpath = "";

if ( ((String)param.get("dirpath")).equals("/") ) {
  // 경우 1: "/" 입력 → 베이스 경로 그대로 사용
  dirpath = basepath;
  // 결과: "/home/ecams/KB001/J001/MyJob"
}
else{
  if( ((String)param.get("dirpath")).indexOf(basepath) == 0 ) {
    // 경우 2: 상대경로가 이미 basepath로 시작
    dirpath = (String)param.get("dirpath");
    // 결과: 입력값 그대로 사용
    // 예: "/home/ecams/KB001/J001/MyJob/src/main/java/com"
  } else {
    // 경우 3: basepath + 상대경로 조합

    // 결과: "/home/ecams/KB001/J001/MyJob" + "/" + "src/main/java/com"

// 더블 슬래시 제거
nparam.put("cm_dirpath", dirpath.replaceAll("//", "/"));
```

## 3. 상대경로(dirpath)와 베이스경로(basepath) 구성 예시

### 예시: 수협은행 시스템에서 Java 프로그램 등록

**입력 정보**:
```
시스템 코드(syscd): "SUHYUP01"  (수협은행)
업무 코드(jobcd):   "JOB001"    (계정 관리)
프로그램명(rsrcname): "AccountService.java"
상대경로(dirpath): "src/main/java/com/bank/service"
프로그램 종류(rsrccd): "R01" (Java 소스)
```

**단계별 처리**:

1. **베이스 경로 조회** (PathService.getPathCD → pathDAO.getBasePath)
   ```sql
   SELECT SCMOWN.SYSHOMEPATH('SUHYUP01', 'R01') || '/' || cm_jobname
   FROM cmm0102
   WHERE cm_jobcd = 'JOB001'
   ```
   결과: `/home/bank/suhyup/JOB001`

2. **경로 조합**
   ```
   basepath = "/home/bank/suhyup/JOB001"
   dirpath(입력) = "src/main/java/com/bank/service"
   dirpath(조합) = "/home/bank/suhyup/JOB001/src/main/java/com/bank/service"
   ```

3. **경로 코드 생성**
   ```
   getMaxDsnCD() → "0000001" (최초 경로)
   ```

4. **DB 저장**
   ```sql
   -- cmm0070: 경로 정보
   INSERT INTO cmm0070 
   VALUES ('SUHYUP01', '0000001', '/home/bank/suhyup/JOB001/src/main/java/com/bank/service', ...);
   
   -- cmm0072: 경로-리소스 매핑
   INSERT INTO cmm0072 
   VALUES ('SUHYUP01', '0000001', 'R01');
   
   -- cmm0073: 경로-업무 매핑
   INSERT INTO cmm0073 
   VALUES ('SUHYUP01', '0000001', 'JOB001');
   
   -- cmr0020: 프로그램 정보
   INSERT INTO cmr0020 
   VALUES ('SUHYUP01', '0000001', 'AccountService.java', 'R01', 'JOB001', 'LIB', ...);
   ```

## 4. 프로그램 종류 하드코딩 여부 검증

### ✅ 결론: **절대 하드코딩되지 않음**

**증거 1: 동적 조회 (FileService.java 1226줄)**
```java
List<ResourceType> rsuList = 
  (List<ResourceType>) resourceTypeDAO.getRsrcInfo(newList.get("syscd").toString());
// DB에서 시스템별 프로그램 종류 조회
```

**증거 2: 확장자 기반 판단 (1229~1239줄)**
```java
// 코드에는 ".java", ".sql" 같은 확장자 정보가 없음
// 모두 ResourceType 테이블의 resourceExeName 컬럼에서 조회
String[] exename = rsuList.get(j).getResourceExeName().split(",");
```

**증거 3: 메서드 서명 분석**
- `ResourceTypeDAO.getRsrcInfo(syscd)`: 매개변수로 시스템 코드를 받음
- 결과적으로 같은 확장자(.java)라도 **시스템마다 다른 프로그램 종류**로 매핑 가능

### 🔧 신규 프로그램 종류 추가 방법

**방법 1: ResourceType 테이블에 레코드 추가** (권장)
```sql
INSERT INTO resourcetype_table 
VALUES (
  'R99',              -- 리소스 코드
  'Python Source',    -- 리소스명
  '.py',              -- 확장자
  '파이썬 소스',      -- 설명
  'PY'                -- 약어
);
```

**방법 2: 기존 프로그램 종류의 확장자 확대**
```sql
UPDATE resourcetype_table 
SET resourceExeName = '.java,.class,.jar,.properties'
WHERE rsrccd = 'R01';
```

**코드 수정은 불필요** (getPathCD, registAllCheckInFile 등 기존 로직 그대로 사용)

## 5. 경로 구성 흐름도

```
플러그인 입력
    ↓
+---+---+---+---+
| syscd | jobcd | dirpath | pgmtype |
+---+---+---+---+
    ↓
    ├─→ [ResourceType 조회] → rsrccd 자동 결정
    │   (확장자 기반, 하드코딩 ✗)
    │
    ├─→ [getBasePath()] → 베이스경로 조회
    │   결과: /home/bank/suhyup/JOB001
    │
    └─→ [경로 조합]
        IF dirpath == "/" → basepath 사용
        ELSE IF dirpath 시작부분 == basepath → dirpath 그대로
        ELSE → basepath + "/" + dirpath
        
        결과: /home/bank/suhyup/JOB001/src/main/java/...
        
        ↓
        [경로코드 할당] → dsncd = "0000001"
        
        ↓
        [DB 저장] → cmm0070/0072/0073 INSERT
        
        ↓
        [프로그램 정보 등록] → cmr0020 INSERT
```

## 6. 핵심 정리

| 항목 | 방식 | 하드코딩 | 비고 |
|------|------|---------|------|
| 프로그램 종류 결정 | ResourceType 테이블 조회 | ✗ | 확장자 기반 동적 판단 |
| 베이스 경로 | DB 함수 SYSHOMEPATH() + jobname | ✗ | 시스템/업무별 동적 조회 |
| 상대경로 | 플러그인에서 수신 | ✗ | 입력값 그대로 사용 |
| 경로 코드 | 시퀀스 자동 할당 | ✗ | getMaxDsnCD() 사용 |
| 경로 조합 | basepath + "/" + dirpath | - | 3가지 로직 분기 |
