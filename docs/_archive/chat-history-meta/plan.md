# 대화기록 메타 영속화 (응답시간·고객사) + 고객사 소프트 락

## 배경
- 답변 하단 뱃지(모델·⏱️소요초·토큰)는 스트리밍 중 DOM 에만 추가되고 `messages` 에는 `{role, content}` 만 저장됨. 새로고침/대화전환 시 재렌더되면서 사라짐.
- 대화 객체에 어느 고객사(repo) 기준 답변인지 정보가 없음.
- 한 대화창에서 여러 고객사를 섞으면 검색 컨텍스트가 오염됨.

## 목표
1. 답변마다 메타(모델·소요초·토큰·고객사)를 `messages` 에 저장 → 재렌더 시 뱃지 복원.
2. 답변 뱃지에 `🏢 고객사명` 표시 (메시지 단위). 히스토리 오가며 이어 물어도 출처가 보임.
3. 소프트 락. 진행 중 대화와 다른 고객사 repo 를 선택하고 전송하면 "새 대화로 시작할까요?" 경고. 확인=새 대화, 취소=현재 대화 계속(강제 아님).

## 범위
- 프론트엔드 단독 (`public/index.html`). 서버/localStorage 스키마는 메시지 객체에 `meta` 필드 추가뿐.
- 스트리밍 경로 + 캐시 응답 경로 둘 다 적용.

## 비범위
- 서버 측 대화 저장소 없음 (chatHistory 는 localStorage).
- 기존 메타 없는 대화는 하위호환(뱃지 없이 본문만, 락 미적용).
