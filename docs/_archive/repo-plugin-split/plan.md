# 레파지토리 플러그인(C) 통합 분리 계획

## 목표
`ecams-ai` 프로젝트의 "레파지토리 통합 관리" 기능에서, 단일로 존재하던 `plugin_client` 타입을 고객사/플랫폼별로 구분(6종)하여 등록하고 관리할 수 있도록 개선.

## 작업 내역
1. UI 모달 렌더링 (`public/index.html`) 내 저장 타입 옵션 분리.
2. 백엔드 Wiki & Graph 빌더 로직(`wikiBuilder.js`, `graphifyBuilder.js`)이 확장된 `plugin_` 접두어 타입을 올바르게 플러그인 타입으로 인식하게 수정.
3. 기존 JSON DB(`repos.json`) 내 타입 일괄 변환(옵션, 검증 위함).
4. 백그라운드 인덱스/위키 생성 시에도 에러 없이 동작하는지 테스트 스크립트로 검증.
