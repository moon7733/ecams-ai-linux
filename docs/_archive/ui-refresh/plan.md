# UI 리프레시 — 모던 미니멀 + 다크모드 토글

## 목표

`public/index.html` (단일 4794줄 SPA) 의 전체 룩앤필을 **모던 미니멀(Linear/Vercel 풍)** 으로 정제하고 **라이트/다크 테마 토글** 을 추가한다. 색 베이스(따뜻한 그레이 + 블루 액센트)는 유지하되 라이트를 1순위로 둔다.

## 왜

현재 사이드바는 컬러 이모지 + 색색의 보더 버튼 14개가 쌓여 산만하고, 이모지가 통일감 없으며, 보더 과다로 답답하다. 전문 SaaS 느낌이 약하다.

## 핵심 제약

- **검증 필수.** 헤드리스 캡처 루프 확보됨 (`docs/ui-refresh/shot.js`, 서버는 포트 5000 상시 기동, 로그인 오버레이는 캡처 시 숨김). 모든 단계 전후로 라이트+다크 스크린샷 비교.
- **단일 파일 = blast radius 큼.** 단계별로 커밋하고, 각 단계는 독립적으로 검증 가능해야 한다.
- **"항상 다크인 영역" 존재.** 코드뷰어/위키/일부 메시지 블록에 다크 배경색(`#0d2137`, `#1e2a3a` 등)이 이미 하드코딩돼 있다. 테마 따라가는 영역과 구분한다.

## 단계 (각 단계 끝에 커밋 + 스크린샷)

1. **토큰화 — 시각 무변화.** 하드코딩 색을 전부 CSS 변수로. 값은 현재 색과 동일. 성공 기준: 라이트 모드 픽셀 동일.
2. **다크 팔레트 + 토글.** `[data-theme="dark"]` 정의, 토글 버튼(localStorage + `prefers-color-scheme` 초기값, FOUC 가드). accent/시맨틱 색은 다크 전용 변형.
3. **이모지 → SVG 아이콘 (~21개).** 일관된 stroke 아이콘 세트(lucide 풍), 단색 `currentColor`.
4. **간격/타이포 폴리시.** 보더 절제, 여백 정리, 버튼 위계 재정립("새 대화"만 강조, 나머지 ghost).

## 비범위

- JS 로직/기능 변경 없음. 마크업은 스타일 정리에 필요한 최소만.
- 모달 내부 전면 재설계는 하지 않음 (토큰화 + 다크 대응까지만).
