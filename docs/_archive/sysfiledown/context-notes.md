# 시스템별 파일 다운로드 컨텍스트 노트 (결정 기록)

### 1. 다운로드 파일 추출 방식 결정
- **배경:** 기존 C 바이너리(`FileRead_cmr0025`)를 쉘로 호출하여 추출할지, 순수 Java로 DB를 조회할지 결정 필요.
- **결정:** 순수 Java 방식(방식 B)으로 DB에서 직접 파일을 읽어 스트리밍 다운로드.
- **사유:** 사용자가 Java에서 직접 수행하는 것이 속도 면에서 유리할 것이라 판단 및 요청함.

### 2. 비동기 호출 방식
- **배경:** 기존 `CheckOut.js` 등에서는 `ajaxCallWithJson`을 사용하여 트리/그리드 데이터를 렌더링.
- **결정:** 비동기 성능 향상을 위해 `ajaxAsync` 공통 함수를 사용하여 렌더링.
- **사유:** 사용자의 구체적 요구사항(ajaxCallWithJson 대신 ajaxAsync 사용).
